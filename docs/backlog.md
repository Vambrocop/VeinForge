# VeinForge 待办 / 受限 / 已知问题日志 (Backlog & Known Limitations)

> **诚实记录"什么没做、为什么、将来从哪接着干"。** 现在受 **CPU + 无小麦/大麦数据** 限制,很多是权宜之计。

> 约定:每次有新硬件/数据/时间,先看本表"受限"与"未建"两节。

## ✅ 已完成(在 main,测试全绿)
- 经典 CV 全链路(io→preprocess→segment→skeleton→measure→report/db)、CLI、napari 查看。
- 性状:密度/脉宽/脉间距(距离图众数法)/自由末端/areole/**纵横脉分离**/整叶形态。
- 胁迫表型:统计(compare,不训练)+ 随机森林(train/predict)。
- DL:可插拔 `DLSegmenter`、U-Net、训练脚本(**clDice + 增强 + 固定 holdout**)。
- 数据/迁移:LeafVeinCNN 真图训练、STARE 视网膜预训练底座;合成 + 真实感模拟器。
- benchmark 记分表(IoU/Dice/速度)+ 多维对比。

## ⚠️ 因 CPU 算力做了妥协(将来用 GPU 撤掉)
| 妥协 | 现状 | 应改为 | 影响 |
|---|---|---|---|
| 图像缩到 128px | 缩图训练 | **原分辨率切块(tiling)** | 细脉丢,准确度被压 |
| 小 U-Net(base32) | 够 CPU 跑 | nnU-Net / 更深 | 容量受限 |
| 训练轮数少(30–40) | 时间所限 | 几百轮 + 早停 | 欠训 |
| 几乎没调参 | 用默认 | lr/loss 权重/优化器搜索 | 未到最优 |
| 无 TTA / 集成 | 单模型单次 | 推理增强 + 集成 | 漏一点精度 |

## ⚠️ 因"没有小麦/大麦数据"未完成(等用户的图)
- **没有真正的小麦/大麦 DL 模型**——现在都是双子叶公共数据 + 合成。
- **纵/横脉分离只在合成图验证**,未在真实单子叶图验证。
- **胁迫表型只在合成数据验证**(随机森林 demo ~98%),无真实胁迫/对照样本。
- 迁移("视网膜帮不帮叶脉")正用修复版重跑;真正有用的迁移要落到小麦大麦。

## 🔲 想做但还没建(功能缺口)
- **ROI / 去直尺**(leaf.jpg 把刻度也当脉描了)。
- **自监督(SSL)** 在大规模无标注叶图上预训练。
- **超分辨**(低分→恢复细脉)。
- **气孔检测/计数、病斑分割、横切解剖(维管束/木质部)** 等其它表型模块。
- **napari 人工校正画笔**(P1.5 计划过,未做)。
- **束鞘排除、脉级(1°–7°)细化**。
- **Streamlit/网页 UI**(非专家用;计划放最后)。
- **主动学习**(让模型挑最该标的图)。
- **与 ImageJ/GrasVIQ/phenoVein 真·同台跑**(现在只用它们发表数字参照,未实跑)。

## 🐞 已知问题(已修 / 待修)
- ✅ **STARE 掩膜极性 bug**:不同图编码不一致,曾把背景当血管 → 已修(占比>0.5 取反),旧 retinal 模型作废、重跑中。
- ✅ **跨轮 IoU 不可比**:早期每轮重切 val → 已用 `--val` 固定 holdout 解决;但历史 0.37/0.53/0.48 那几个数**不可直接比**。
- 🔲 **切块管线未实现**:目前只缩图;tiling 待做。
- 🔲 **DB schema 无迁移**:新增列只对新建库生效,老 `veinforge.db` 不自动加列(研究期可接受)。
- 🔲 **DL 训练与推理预处理一致性**:需保证两端同样处理(原图 or 同样 preprocess),否则掉点。

## ▶️ 从哪接着干(优先级)
1. **拿到小麦/大麦图** → P1 自动描 + Ilastik 修 → 微调(最高价值)。
2. **上免费云 GPU** → 原分辨率切块 + 大模型 + 多轮(撤掉上面的 CPU 妥协)。
3. ROI/去直尺、纵横脉真实验证、真·同台 benchmark。
