# VeinForge 训练优化计划 (Training Optimization Plan)

> **给未来的我们**:等有了 GPU / 更多标注数据 / 更多预算,照这份把训练做到位。
> 现在的一切(128px、小 U-Net、少轮)都是 **CPU + 小数据** 下的权宜之计。
> 配套:[`benchmark.md`](benchmark.md)(先比较,看短板)。

## 0. 总原则:准确度第一,且要"诚实的准确度"
固定 holdout、按叶片分(无泄漏)、参数可复现;**绝不**靠缩小验证集 / 挑数据 / 调阈值制造好看数字。`train_dl.py --val <固定holdout>` 已按最佳 val IoU 存模型。

## 1. 现状与瓶颈(按影响排序)
1. **数据**(最大瓶颈):还没有小麦/大麦标注脉图。数据决定准确度**天花板**。
2. **算力**(第二):CPU 逼着用 128px 缩图、小模型、少轮、少试 → 压住了能逼近天花板的程度。
3. **已做到**:经典 CV(IoU 0.52);DL 严谨化(clDice+增强)把叶脉 DL 0.48→**0.66**,已反超经典。

## 2. 提升准确度的"杠杆"(按性价比排序)
| 杠杆 | 现在 | 有资源后 | 预期收益 |
|---|---|---|---|
| **数据量/质量** | 几百 tile(双子叶公共) | + 你的小麦大麦 + 更多公共 | ⭐ 最大 |
| **训练方法** | clDice+增强 ✓ | + 更强增强、正则、调度 | 已见效,继续 |
| **分辨率** | 128px **缩图** | **原分辨率切块(tiling)** | 中–大(细脉回来) |
| **模型** | 小 U-Net(base32) | **nnU-Net** / 注意力/残差 U-Net | 中 |
| **迁移/自监督** | STARE 视网膜 | 视网膜 + **大规模无标注叶图 SSL** 预训练 | 中 |
| **调参/损失权重** | 默认 | 学习率/clDice 权重/优化器搜索 | 小–中 |
| **推理增强** | 无 | TTA(翻转旋转平均)+ 模型集成 | 小 |

> 记住排序:**数据 > 方法 > 分辨率 ≈ 模型 > 调参 > 推理增强**。别本末倒置去先堆大模型。

## 3. 算力计划:免费云 GPU(不用买硬件)
- **Colab / Kaggle 送免费 GPU**。配方:repo + 数据上传 → `pip install -e ".[dl]"` → 同一套 `train_dl.py`,只是把 `--size` 调到 256/512、`--epochs` 调大、UNet `base` 调大。
- GPU 解锁的(CPU 上做不动的):**原分辨率、大模型、几百轮、批量调参**。
- 注意:GPU **只让训练更快/更能放开手脚**,不直接改准确度;但"放开手脚"(原图+大模型+多轮)本身会显著提准确度。

## 4. 数据计划:怎么省力拿到标注(准确度的天花板)
1. **你的小麦/大麦透明化图** → 用 **P1 经典 CV 自动描一版** → 在 **Ilastik** 里人工修几笔 → 当训练真值(零从头手画)。
2. **主动学习**:先标少量→训→让模型挑"最不确定"的图给你**优先标**,标注效率最大化。
3. **公共预训练**:视网膜(STARE/DRIVE)+ 大规模**无标注**叶图**自监督(SSL)** → 通用底座,再用你的图微调。

## 5. 严谨评估(让"提升"可信)
- 固定 holdout(按叶片,无泄漏)—— `--val` 已实现。
- **多维记分表**(见 benchmark.md):准确度(IoU/Dice/clDice)、速度、信息完整度、自动化、可复现、鲁棒性、拓扑正确性、物种适用、成本、质控。
- 对照 **ImageJ 手动 / GrasVIQ / phenoVein**。
- 每次"提升"都要在**同一个固定 holdout**上比,否则数字不可比(我们踩过这个坑)。

## 6. "有 GPU + 数据后,照这个序列跑"
1. **切块**:原分辨率 256/512 tile(替代 128 缩图),保住细脉 + 多出样本。
2. **公共预训练**:retinal + 无标注叶 SSL → `base.pt`。
3. **微调**:`train_dl.py --data data/barley --init base.pt --val <holdout> --size 512 --epochs 200`(GPU 上可行)。
4. **选最佳**(by val)→ 加 **TTA + 集成**。
5. **多维 benchmark** vs ImageJ/GrasVIQ → 找最弱维 → 针对性迭代(回到第 1 步)。

## 7. 诚实的预期
- **数据是天花板,方法/算力帮你逼近它**。别指望单独某一招翻倍;是"数据 × 方法 × 算力"叠加。
- 当前 CPU + 小数据下 **IoU≈0.66 已不错**;有 GPU + 你的标注数据,目标**对标 GrasVIQ ~0.9+**。
- 任何"提升"都以**固定 holdout 上的真实数字**为准,不接受好看但不可比的数。
