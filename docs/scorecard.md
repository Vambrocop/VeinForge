# VeinForge 项目自评记分表 (Project Self-Scorecard)

> 给**项目本身**多维打分(不是 vs ImageJ 的技术对比,那在 [`benchmark.md`](benchmark.md))。
> 用途:**看哪强哪弱 → 针对性提升 → 升版本时重打分,追踪进步**。
> 打分**诚实**:不自夸,弱项如实给低分。满分 10。

## 当前评分 — v0.1.0（2026-06-15）

| 维度 | 分 | 依据 | 怎么提分 |
|---|---:|---|---|
| **专业性 / 严谨** | 9 | TDD、42 测试全绿、固定 holdout、文献支撑、主动报 bug | 真·同台 benchmark、更多真实验证 |
| **设计 / 架构** | 9 | 模块化、可插拔 `segmenter`(经典↔DL 同接口)、清晰边界 | tiling 管线、插件化、UI 解耦 |
| **快速性** | 9 | 0.04–0.09 s/图,碾压 ImageJ 手描(分钟级) | GPU 推理(其实已够快) |
| **可复现性** | 9 | 参数快照 + SQLite + 固定 holdout + 确定性 | DB schema 迁移 |
| **文档** | 9 | spec / plan / roadmap / backlog / ideas / benchmark 齐全 | 随项目持续更新 |
| **完整性** | 8 | P1 全链路 + P2 框架 + 胁迫表型 + benchmark | 补 UI、未建模块(气孔/病斑/超分) |
| **新颖性** | 7 | 胁迫表型接口、单子叶纵/横脉分离、整合度 | **落地胁迫真新意**、SSL、超分辨 |
| **准确性** | 6 | 叶脉 IoU 0.52(经典)→0.66(DL);落后 GrasVIQ 自报 ~0.9 | **数据 + GPU + 原分辨率 + nnU-Net**(见 training-plan) |
| **可用性** | 6 | CLI / napari 好,但**无非专家 GUI** | napari 校正画笔、Streamlit 网页 |
| **数据成熟度** | 3 | 无小麦/大麦标注,双子叶公共数据为主 | **用户的透明化脉图(最高优先)** |

**综合 ≈ 7.5 / 10**(简单平均)。

## 一句话画像
**工程与设计强、快、可复现、文档全;短板集中在"准确度 / 可用性 / 数据成熟度",而这三者几乎都卡在同一处——缺小麦/大麦数据 + GPU + 一个面向非专家的 UI。** 提分路径已写在 [`training-plan.md`](training-plan.md)。

## 提分优先级(性价比)
1. 🔴 **数据成熟度 3→**:拿到小麦/大麦图(P1 自动描 + Ilastik 修)→ 直接拉高准确性 + 新颖性。
2. 🟠 **准确性 6→**:免费云 GPU + 原分辨率切块 + nnU-Net(撤掉 CPU 妥协)。
3. 🟡 **可用性 6→**:napari 人工校正 → Streamlit 网页。
